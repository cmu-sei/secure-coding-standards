# Runtime Environment (ENV)

## Rules

-   [ENV00-J. Do not sign code that performs only unprivileged operations](/sei-cert-oracle-coding-standard-for-java/rules/runtime-environment-env/env00-j)
-   [ENV01-J. Place all security-sensitive code in a single JAR and sign and seal it](/sei-cert-oracle-coding-standard-for-java/rules/runtime-environment-env/env01-j)
-   [ENV02-J. Do not trust the values of environment variables](/sei-cert-oracle-coding-standard-for-java/rules/runtime-environment-env/env02-j)
-   [ENV03-J. Do not grant dangerous combinations of permissions](/sei-cert-oracle-coding-standard-for-java/rules/runtime-environment-env/env03-j/)
-   [ENV04-J. Do not disable bytecode verification](/sei-cert-oracle-coding-standard-for-java/rules/runtime-environment-env/env04-j)
-   [ENV05-J. Do not deploy an application that can be remotely monitored](/sei-cert-oracle-coding-standard-for-java/rules/runtime-environment-env/env05-j)
-   [ENV06-J. Production code must not contain debugging entry points](/sei-cert-oracle-coding-standard-for-java/rules/runtime-environment-env/env06-j)
