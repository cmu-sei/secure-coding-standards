# JNI02-J. Do not assume object references are constant or unique

# (THIS CODING RULE OR GUIDELINE IS UNDER CONSTRUCTION)

According to \[ [JNI Tips](https://www.securecoding.cert.org/confluence/display/java/AA.+References#AA.References-JNItips) \], section "Local and Global References", references in native code to the same object may have different values. Return values from the `NewGlobalRef()` function when applied to the same object may differ from each other. Consequently, object references are not necessarily constant or unique. Object references should never be compared using == or `!=` in native code. When testing for object equality, the `IsSameObject()` function should be used instead of `==` .

## Risk Assessment

If it is assumed that an object reference is constant or unique then erroneous results may be obtained that could lead to the app crashing. This, in turn, could be used to mount a denial-of-service attack.
