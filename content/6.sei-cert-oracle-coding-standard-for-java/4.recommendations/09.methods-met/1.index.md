# Methods (MET)

-   [MET50-J. Avoid ambiguous or confusing uses of overloading](/sei-cert-oracle-coding-standard-for-java/recommendations/methods-met/met50-j)
-   [MET51-J. Do not use overloaded methods to differentiate between runtime types](/sei-cert-oracle-coding-standard-for-java/recommendations/methods-met/met51-j)
-   [MET52-J. Do not use the clone() method to copy untrusted method parameters](/sei-cert-oracle-coding-standard-for-java/recommendations/methods-met/met52-j)
-   [MET53-J. Ensure that the clone() method calls super.clone()](/sei-cert-oracle-coding-standard-for-java/recommendations/methods-met/met53-j)
-   [MET54-J. Always provide feedback about the resulting value of a method](/sei-cert-oracle-coding-standard-for-java/recommendations/methods-met/met54-j)
-   [MET55-J. Return an empty array or collection instead of a null value for methods that return an array or collection](/sei-cert-oracle-coding-standard-for-java/recommendations/methods-met/met55-j)
-   [MET56-J. Do not use Object.equals() to compare cryptographic keys](/sei-cert-oracle-coding-standard-for-java/recommendations/methods-met/met56-j)
