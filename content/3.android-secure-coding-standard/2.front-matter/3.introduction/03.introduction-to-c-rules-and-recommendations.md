# Introduction to C Rules and Recommendations

Some of the CERT C Coding Standard's rules and recommendations are applicable to developing Android applications, as native code. Our latest published edition of rules and recommendations for C is a free and publicly-available downloadable document in the [CERT C Coding Standard, 2016 Edition](http://www.cert.org/secure-coding/products-services/secure-coding-download.cfm) , and our latest published edition including C++ is the [<u>Secure Coding in C and C++, 2nd Edition</u>](http://www.informit.com/store/secure-coding-in-c-and-c-plus-plus-9780321822130) *.* Development of the standards continues on this [<u>wiki</u>](https://www.securecoding.cert.org/confluence/display/seccode/CERT+C+Coding+Standard) . Not all of the coding standard is applicable to the Android environment. We have performed an initial analysis of Android applicability of the C coding rules and recommendations, [here](https://www.securecoding.cert.org/confluence/pages/viewpage.action?pageId=134807610) . The analysis is presented in a chart form, and each rule or recommendation listed in the chart is hyperlinked to its full wiki page. For each C rule or recommendation found to require Android-specific implementation advice, that advice is provided in a column of the analysis chart.
