# PRE04-C. Do not reuse a standard header file name

If a file with the same name as a standard header is placed in the search path for included source files, the behavior is [undefined](/sei-cert-c-coding-standard/back-matter/bb-definitions#BB.Definitions-undefinedbehavior) .

The following table from the C Standard, subclause 7.1.2 \[ [ISO/IEC 9899:2011](/sei-cert-c-coding-standard/back-matter/aa-bibliography#AA.Bibliography-ISO-IEC9899-2011) \], lists these standard headers:

Do not reuse standard header file names, system-specific header file names, or other header file names.

## Noncompliant Code Example

In this noncompliant code example, the programmer chooses to use a local version of the standard library but does not make the change clear:

::code-block{quality="bad"}
``` c
#include "stdio.h"  /* Confusing, distinct from <stdio.h> */

/* ... */
```
::

## Compliant Solution

The solution addresses the problem by giving the local library a unique name (per [PRE08-C. Guarantee that header file names are unique](/sei-cert-c-coding-standard/recommendations/preprocessor-pre/pre08-c) ), which makes it apparent that the library used is not the original:

::code-block{quality="good"}
``` c
/* Using a local version of stdio.h */ 
#include "mystdio.h"

/* ... */
```
::

Using header file names that conflict with other header file names can result in an incorrect file being included.
