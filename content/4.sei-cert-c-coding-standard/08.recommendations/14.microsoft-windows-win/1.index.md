# Microsoft Windows (WIN)

Warning

This section is under construction.

This appendix contains guidelines for functions that are defined as part of the Microsoft Windows API \[ [MSDN](http://msdn.microsoft.com/en-us/library/windows/desktop/hh447209(v=vs.85).aspx) \] and the Microsoft Visual C++ compiler \[ [MSDN](http://msdn.microsoft.com/en-us/library/vstudio/60k1461a.aspx) \]. These rules and recommendations are not part of the core standard because they do not apply in all C language applications. The intent of providing these guidelines is to demonstrate how rules and recommendations for other standards or specific [BB. Definitions#implementation](/sei-cert-c-coding-standard/back-matter/bb-definitions#BB.Definitions-implementation) may be integrated with the core C recommendations.

-   [WIN00-C. Be specific when dynamically loading libraries](/sei-cert-c-coding-standard/recommendations/microsoft-windows-win/win00-c)
-   [WIN01-C. Do not forcibly terminate execution](/sei-cert-c-coding-standard/recommendations/microsoft-windows-win/win01-c)
-   [WIN02-C. Restrict privileges when spawning child processes](/sei-cert-c-coding-standard/recommendations/microsoft-windows-win/win02-c)
-   [WIN03-C. Understand HANDLE inheritance](/sei-cert-c-coding-standard/recommendations/microsoft-windows-win/win03-c)
-   [WIN04-C. Consider encrypting function pointers](/sei-cert-c-coding-standard/recommendations/microsoft-windows-win/win04-c)
