# POSIX (POS)

This appendix contains guidelines for functions that are defined as part of the POSIX family of standards but are not included in the C Standard. These rules and recommendations are not part of the core standard because they do not apply in all C language applications and because they represent an incomplete set. The intent of providing these guidelines is to demonstrate how rules and recommendations for other standards or specific implementations may be integrated with the core C recommendations.

-   [POS30-C. Use the readlink() function properly](/sei-cert-c-coding-standard/rules/posix-pos/pos30-c)
-   [POS34-C. Do not call putenv() with a pointer to an automatic variable as the argument](/sei-cert-c-coding-standard/rules/posix-pos/pos34-c)
-   [POS35-C. Avoid race conditions while checking for the existence of a symbolic link](/sei-cert-c-coding-standard/rules/posix-pos/pos35-c)
-   [POS36-C. Observe correct revocation order while relinquishing privileges](/sei-cert-c-coding-standard/rules/posix-pos/pos36-c)
-   [POS37-C. Ensure that privilege relinquishment is successful](/sei-cert-c-coding-standard/rules/posix-pos/pos37-c)
-   [POS38-C. Beware of race conditions when using fork and file descriptors](/sei-cert-c-coding-standard/rules/posix-pos/pos38-c)
-   [POS39-C. Use the correct byte ordering when transferring data between systems](/sei-cert-c-coding-standard/rules/posix-pos/pos39-c)
-   [POS44-C. Do not use signals to terminate threads](/sei-cert-c-coding-standard/rules/posix-pos/pos44-c)
-   [POS47-C. Do not use threads that can be canceled asynchronously](/sei-cert-c-coding-standard/rules/posix-pos/pos47-c)
-   [POS48-C. Do not unlock or destroy another POSIX thread's mutex](/sei-cert-c-coding-standard/rules/posix-pos/pos48-c)
-   [POS49-C. When data must be accessed by multiple threads, provide a mutex and guarantee no adjacent data is also accessed](/sei-cert-c-coding-standard/rules/posix-pos/pos49-c)
-   [POS50-C. Declare objects shared between POSIX threads with appropriate storage durations](/sei-cert-c-coding-standard/rules/posix-pos/pos50-c)
-   [POS51-C. Avoid deadlock with POSIX threads by locking in predefined order](/sei-cert-c-coding-standard/rules/posix-pos/pos51-c)
-   [POS52-C. Do not perform operations that can block while holding a POSIX lock](/sei-cert-c-coding-standard/rules/posix-pos/pos52-c)
-   [POS53-C. Do not use more than one mutex for concurrent waiting operations on a condition variable](/sei-cert-c-coding-standard/rules/posix-pos/pos53-c)
-   [POS54-C. Detect and handle POSIX library errors](/sei-cert-c-coding-standard/rules/posix-pos/pos54-c)
