# PRE33-C. Do not suppress the macro definitions of stdatomic.h or stdbit.h generic functions

The C preprocessor allows suppressing macro definitions to access an actual function with the same name. This must not be done to generic functions declared in `stdatomic.h` or `stdbit.h`. Doing so results in [undefined behavior](/sei-cert-c-coding-standard/back-matter/bb-definitions#BB.Definitions-undefinedbehavior) , according to the C Standard, 7.17.1, paragraph 7, and 7.18.1, paragraph 5 \[ [ISO/IEC 9899:2024](/sei-cert-c-coding-standard/back-matter/aa-bibliography#AA.Bibliography-ISO-IEC9899-2024) \]:

> It is unspecified whether any generic function declared in <stdatomic.h> is a macro or an identifier declared with external linkage. **If a macro definition is suppressed to access an actual function, or a program defines an external identifier with the name of a generic function, the behavior is undefined.**

> It is unspecified whether any generic function declared in <stdbit.h> is a macro or an identifier declared with external linkage. **If a macro definition is suppressed to access an actual function, or a program defines an external identifier with the name of a generic function, the behavior is unspecified.**

See also [undefined behavior 145](/sei-cert-c-coding-standard/back-matter/cc-undefined-behavior#CC.UndefinedBehavior-ub_145) .

## Noncompliant Code Example

In this noncompliant code example, the programmer wraps the generic `atomic_store_explicit` function in parentheses, which suppresses any macro definition of it.

::code-block{quality="bad"}
``` c
#include <stdatomic.h>

void func(atomic_int *i) {
  (atomic_store_explicit)(i, 42, memory_order_relaxed);
}
```
::

## Compliant Solution

In this compliant solution, there are no parentheses around `atomic_store_explicit`, so a macro definition of it will be used if it was implemented as one.

::code-block{quality="good"}
``` c
#include <stdatomic.h>

void func(atomic_int *i) {
  atomic_store_explicit(i, 42, memory_order_relaxed);
}
```
::

## Risk Assessment

Suppressing the macro definitions of stdatomic.h or stdbit.h generic functions is [undefined behavior 145](/sei-cert-c-coding-standard/back-matter/cc-undefined-behavior#CC.UndefinedBehavior-ub_145) .
