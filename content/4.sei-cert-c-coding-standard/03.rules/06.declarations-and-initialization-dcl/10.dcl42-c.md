# DCL42-C. Only call functions with the unsequenced or reproducible attributes if they actually have the asserted property

The function attribute `reproducible` means that a function is effectiveless and idempotent. The function attribute `unsequenced` means that a function is stateless, effectless, idempotent and independent.

It is the programmer's responsibility to ensure that functions with these attributes actually meet all of those criteria. If they do not, calling them results in [undefined behavior](/sei-cert-c-coding-standard/back-matter/bb-definitions#BB.Definitions-undefinedbehavior) , according to the C Standard, 6.7.13.8.1, paragraph 3 \[ [ISO/IEC 9899:2024](/sei-cert-c-coding-standard/back-matter/aa-bibliography#AA.Bibliography-ISO-IEC9899-2024) \]:

> if a definition that does not have the asserted property is accessed by a function declaration or a function pointer with a type that has the attribute, the behavior is undefined.

See also [undefined behavior 83](/sei-cert-c-coding-standard/back-matter/cc-undefined-behavior#CC.UndefinedBehavior-ub_83) .

## Noncompliant Code Example

In this noncompliant code example, the `adjust` function does not have the idempotence property that is asserted by the `reproducible` attribute (repeating a call will result in reducing it by 6 instead of just by 3), so calling it results in undefined behavior.

::code-block{quality="bad"}
``` c
void adjust(unsigned *restrict x) [[reproducible]] {
    *x -= 3;
}
```
::

## Compliant Solution (remove attribute)

In this compliant solution, the `reproducible` attribute has been removed.

::code-block{quality="good"}
``` c
void adjust(unsigned *restrict x) {
    *x &= ~3;
}
```
::

## Compliant Solution (change behavior)

In this compliant solution, the `adjust` operation performs a different operation that is actually idempotent (masking off the same bits twice in a row won't change the value the second time).

::code-block{quality="good"}
``` c
void adjust(unsigned *restrict x) [[reproducible]] {
    *x &= ~3;
}
```
::

## Risk Assessment

Calling functions with the `unsequenced` or `reproducible` attributes that do not actually have the asserted property is [undefined behavior 83](/sei-cert-c-coding-standard/back-matter/cc-undefined-behavior#CC.UndefinedBehavior-ub_83) .
