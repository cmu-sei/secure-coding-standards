# DCL43-C. Do not use auto to declare two or more variables

C23 \[ [ISO/IEC 9899:2024](/sei-cert-c-coding-standard/back-matter/aa-bibliography#AA.Bibliography-ISO-IEC9899-2024) \] added support for type inference using the `auto` keyword.  This new feature comes with [undefined behavior 78](/sei-cert-c-coding-standard/back-matter/cc-undefined-behavior#CC.UndefinedBehavior-ub_78), which covers the case where an `auto` declaration contains 2 or more variables or 0 variables.

## Noncompliant Code Example

This noncompliant code example declares two variables in one declaration using `auto`.

::code-block{quality="bad"}
``` c
auto i = 3, j = 4.5; // Undefined Behavior
```
::

### Implementation Details

In GCC 11.4.0, this line of code declares both i and j as ints, setting j to 4!

## Compliant Solution

In this compliant solution, each declarator is sequestered to its own declaration.

::code-block{quality="good"}
``` c
auto i = 3;
auto j = 4.5;
```
::

## Risk Assessment

Using `auto` to declare zero or two or more variables is undefined behavior [78](/sei-cert-c-coding-standard/back-matter/cc-undefined-behavior#CC.UndefinedBehavior-ub_78).
