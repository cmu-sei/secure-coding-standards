Pages should have tags to indicate the status of their corresponding checker in Compass Rose:

|                                                                                                     |                                                                                                             |
|-----------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------|
| **Tag**                                                                                             | Meaning                                                                                                     |
| [rose-complete](https://www.securecoding.cert.org/confluence/label/rose-complete)                   | ROSE catches all violations                                                                                 |
| [rose-partial](https://www.securecoding.cert.org/confluence/label/rose-partial)                     | ROSE catches some violations                                                                                |
| [rose-possible](https://www.securecoding.cert.org/confluence/label/rose-possible)                   | ROSE could catch some or all violations, but doesn't yet.                                                   |
| [rose-gcc](https://www.securecoding.cert.org/confluence/label/rose-gcc)                             | ROSE doesn't catch violations, but will soon, `GCC` catches violations                         |
| [unenforceable](https://www.securecoding.cert.org/confluence/label/unenforceable)                   | These rules can't be checked automatically.                                                                 |
| [rose-nonapplicable](https://www.securecoding.cert.org/confluence/label/rose-nonapplicable)         | These rules could be checked automatically in theory, but not by ROSE.                                      |
| [rose-na-macros](https://www.securecoding.cert.org/confluence/label/rose-na-macros)                 | ROSE could check these rules if it recognized macro usage.                                                  |
| [rose-na-exptypes](https://www.securecoding.cert.org/confluence/label/rose-na-exptypes)             | ROSE could check these rules if it recognized derived types in expressions, such as `size_t` . |
| [rose-na-multiple-files](https://www.securecoding.cert.org/confluence/label/rose-na-multiple-files) | ROSE could check these rules if it operated on multiple files at once.                                      |
| [rose-false-positive](https://www.securecoding.cert.org/confluence/label/rose-false-positive)       | ROSE could enforce this rule, but could not avoid catching some false positives.                            |

At this point, all rules should have one of these tags. That is, they should be completely or partially checked by ROSE, or they should be marked 'rose-possible', in that we will try to check them with ROSE, or they should have one of the nonapplicable tags indicating we don't think they can be checked with ROSE.
