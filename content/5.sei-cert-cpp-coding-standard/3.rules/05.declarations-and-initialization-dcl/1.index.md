# Declarations and Initialization (DCL)

<div class="columnLayout two-right-sidebar" layout="two-right-sidebar">

<div class="cell normal" data-type="normal">

<div class="innerCell">

-   [DCL50-CPP. Do not define a C-style variadic function](/sei-cert-cpp-coding-standard/rules/declarations-and-initialization-dcl/dcl50-cpp)
-   [DCL51-CPP. Do not declare or define a reserved identifier](/sei-cert-cpp-coding-standard/rules/declarations-and-initialization-dcl/dcl51-cpp)
-   [DCL52-CPP. Never qualify a reference type with const or volatile](/sei-cert-cpp-coding-standard/rules/declarations-and-initialization-dcl/dcl52-cpp)
-   [DCL53-CPP. Do not write syntactically ambiguous declarations](/sei-cert-cpp-coding-standard/rules/declarations-and-initialization-dcl/dcl53-cpp)
-   [DCL54-CPP. Overload allocation and deallocation functions as a pair in the same scope](/sei-cert-cpp-coding-standard/rules/declarations-and-initialization-dcl/dcl54-cpp)
-   [DCL55-CPP. Avoid information leakage when passing a class object across a trust boundary](/sei-cert-cpp-coding-standard/rules/declarations-and-initialization-dcl/dcl55-cpp)
-   [DCL56-CPP. Avoid cycles during initialization of static objects](/sei-cert-cpp-coding-standard/rules/declarations-and-initialization-dcl/dcl56-cpp)
-   [DCL57-CPP. Do not let exceptions escape from destructors or deallocation functions](/sei-cert-cpp-coding-standard/rules/declarations-and-initialization-dcl/dcl57-cpp)
-   [DCL58-CPP. Do not modify the standard namespaces](/sei-cert-cpp-coding-standard/rules/declarations-and-initialization-dcl/dcl58-cpp)
-   [DCL59-CPP. Do not define an unnamed namespace in a header file](/sei-cert-cpp-coding-standard/rules/declarations-and-initialization-dcl/dcl59-cpp)
-   [DCL60-CPP. Obey the one-definition rule](/sei-cert-cpp-coding-standard/rules/declarations-and-initialization-dcl/dcl60-cpp)

  

The following rules from the [SEI CERT C Coding Standard](/sei-cert-c-coding-standard/) also apply in C++:

-   [DCL30-C. Declare objects with appropriate storage durations](/sei-cert-c-coding-standard/rules/declarations-and-initialization-dcl/dcl30-c)
-   [DCL39-C. Avoid information leakage when passing a structure across a trust boundary](/sei-cert-c-coding-standard/rules/declarations-and-initialization-dcl/dcl39-c)
-   [DCL40-C. Do not create incompatible declarations of the same function or object](/sei-cert-c-coding-standard/rules/declarations-and-initialization-dcl/dcl40-c)
