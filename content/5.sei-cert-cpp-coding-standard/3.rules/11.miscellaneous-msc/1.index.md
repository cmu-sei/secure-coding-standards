-   [MSC51-CPP. Ensure your random number generator is properly seeded](/sei-cert-cpp-coding-standard/rules/miscellaneous-msc/msc51-cpp)

The following rules from the [SEI CERT C Coding Standard](/sei-cert-c-coding-standard/) also apply in C++:

-   [MSC30-C. Do not use the rand() function for generating pseudorandom numbers](/sei-cert-c-coding-standard/rules/miscellaneous-msc/msc30-c)
-   [MSC32-C. Properly seed pseudorandom number generators](/sei-cert-c-coding-standard/rules/miscellaneous-msc/msc32-c)
-   [MSC33-C. Do not pass invalid data to the asctime() function](/sei-cert-c-coding-standard/rules/miscellaneous-msc/msc33-c)
-   [MSC37-C. Ensure that control never reaches the end of a non-void function](/sei-cert-c-coding-standard/rules/miscellaneous-msc/msc37-c)
-   [MSC38-C. Do not treat a predefined identifier as an object if it might only be implemented as a macro](/sei-cert-c-coding-standard/rules/miscellaneous-msc/msc38-c)
-   [MSC39-C. Do not call va_arg() on a va_list that has an indeterminate value](/sei-cert-c-coding-standard/rules/miscellaneous-msc/msc39-c)
-   [MSC40-C. Do not violate constraints](/sei-cert-c-coding-standard/rules/miscellaneous-msc/msc40-c)
-   [MSC41-C. Never hard code sensitive information](/sei-cert-c-coding-standard/rules/miscellaneous-msc/msc41-c)

-   [FLP30-C. Do not use floating-point variables as loop counters](/sei-cert-c-coding-standard/rules/floating-point-flp/flp30-c)
-   [FLP32-C. Prevent or detect domain and range errors in math functions](/sei-cert-c-coding-standard/rules/floating-point-flp/flp32-c)
-   [FLP34-C. Ensure that floating-point conversions are within range of the new type](/sei-cert-c-coding-standard/rules/floating-point-flp/flp34-c)
-   [FLP36-C. Preserve precision when converting integral values to floating-point type](/sei-cert-c-coding-standard/rules/floating-point-flp/flp36-c)
-   [FLP37-C. Do not use object representations to compare floating-point values](/sei-cert-c-coding-standard/rules/floating-point-flp/flp37-c)

<!-- -->

-   [ENV30-C. Do not modify the object referenced by the return value of certain functions](/sei-cert-c-coding-standard/rules/environment-env/env30-c)
-   [ENV31-C. Do not rely on an environment pointer following an operation that may invalidate it](/sei-cert-c-coding-standard/rules/environment-env/env31-c)
-   [ENV32-C. All exit handlers must return normally](/sei-cert-c-coding-standard/rules/environment-env/env32-c)
-   [ENV33-C. Do not call system()](/sei-cert-c-coding-standard/rules/environment-env/env33-c)
-   [ENV34-C. Do not store pointers returned by certain functions](/sei-cert-c-coding-standard/rules/environment-env/env34-c)

<!-- -->

-   [SIG31-C. Do not access shared objects in signal handlers](/sei-cert-c-coding-standard/rules/signals-sig/sig31-c)
-   [SIG34-C. Do not call signal() from within interruptible signal handlers](/sei-cert-c-coding-standard/rules/signals-sig/sig34-c)
-   [SIG35-C. Do not return from a computational exception signal handler](/sei-cert-c-coding-standard/rules/signals-sig/sig35-c)

-   [PRE31-C. Avoid side effects in arguments to unsafe macros](/sei-cert-c-coding-standard/rules/preprocessor-pre/pre31-c)
-   [PRE32-C. Do not use preprocessor directives in invocations of function-like macros](/sei-cert-c-coding-standard/rules/preprocessor-pre/pre32-c)
