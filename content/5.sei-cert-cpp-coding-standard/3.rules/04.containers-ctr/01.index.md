# Containers (CTR)

<div class="contentLayout2">

<div class="columnLayout two-right-sidebar" layout="two-right-sidebar">

<div class="cell normal" data-type="normal">

<div class="innerCell">

-   [CTR50-CPP. Guarantee that container indices and iterators are within the valid range](/sei-cert-cpp-coding-standard/rules/containers-ctr/ctr50-cpp)
-   [CTR51-CPP. Use valid references, pointers, and iterators to reference elements of a container](/sei-cert-cpp-coding-standard/rules/containers-ctr/ctr51-cpp)
-   [CTR52-CPP. Guarantee that library functions do not overflow](/sei-cert-cpp-coding-standard/rules/containers-ctr/ctr52-cpp)
-   [CTR53-CPP. Use valid iterator ranges](/sei-cert-cpp-coding-standard/rules/containers-ctr/ctr53-cpp)
-   [CTR54-CPP. Do not subtract iterators that do not refer to the same container](/sei-cert-cpp-coding-standard/rules/containers-ctr/ctr54-cpp)
-   [CTR55-CPP. Do not use an additive operator on an iterator if the result would overflow](/sei-cert-cpp-coding-standard/rules/containers-ctr/ctr55-cpp)
-   [CTR56-CPP. Do not use pointer arithmetic on polymorphic objects](/sei-cert-cpp-coding-standard/rules/containers-ctr/ctr56-cpp)
-   [CTR57-CPP. Provide a valid ordering predicate](/sei-cert-cpp-coding-standard/rules/containers-ctr/ctr57-cpp)
-   [CTR58-CPP. Predicate function objects should not be mutable](/sei-cert-cpp-coding-standard/rules/containers-ctr/ctr58-cpp)

  

The following rules from the [SEI CERT C Coding Standard](/sei-cert-c-coding-standard/) also apply in C++:

-   [ARR30-C. Do not form or use out-of-bounds pointers or array subscripts](/sei-cert-c-coding-standard/rules/arrays-arr/arr30-c)
-   [ARR37-C. Do not add or subtract an integer to a pointer to a non-array object](/sei-cert-c-coding-standard/rules/arrays-arr/arr37-c)
-   [ARR38-C. Guarantee that library functions do not form invalid pointers](/sei-cert-c-coding-standard/rules/arrays-arr/arr38-c)
-   [ARR39-C. Do not add or subtract a scaled integer to a pointer](/sei-cert-c-coding-standard/rules/arrays-arr/arr39-c)
